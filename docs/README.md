## 📋 文档规范

### 文档元信息模板

每个文档必须包含以下元信息（时间戳精确到分钟）：

```markdown
# 文档标题

**文档编号**: XXX-YYY
**创建时间**: YYYY-MM-DD HH:mm
**最后修改**: YYYY-MM-DD HH:mm
**执行模型**: claude-sonnet-4-5 (claude-sonnet-4-5-20250929)
**文档状态**: 草稿 / 评审中 / 已确认 / 生效中

---

## 修改历史

| 日期 | 修改人/模型 | 修改概要 |
|------|------------|---------|
| YYYY-MM-DD HH:mm | claude-sonnet-4-5 | 创建文档 |

---

[文档正文]
```

### 文档命名规范

- 使用小写字母和连字符
- 英文文档名（如 `code-patterns.md`）
- 或中文大写标题（如 `PRODUCT_TRANSFORMATION_PLAN.md`）
- 带序号的文档使用两位数字（如 `01-data-model-refactor.md`）
