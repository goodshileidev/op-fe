## 📊 文档体系架构

### 双文档体系

```
CLAUDES.md（短期记忆）
    ├─ 定位: 开发指南
    ├─ 受众: AI 助手、开发人员
    ├─ 内容: 流程、命令、规范
    └─ 更新: 频繁（跟随任务）

CLAUDE.md（长期记忆）
    ├─ 定位: 技术文档
    ├─ 受众: 架构师、新人培训
    ├─ 内容: 架构、业务、模型
    └─ 更新: 定期（重大变更）
```

### 完整文档层次

```
项目文档/
├── README.md                    # 项目说明书
├── CLAUDES.md ⭐                # 短期记忆（开发指南）
├── CLAUDE.md                    # 长期记忆（技术文档）
├── DOCUMENTATION.md             # 文档导航索引
└── docs/                        # 专题文档目录
    ├── README.md               # docs 导航
    ├── PRODUCT_TRANSFORMATION_PLAN.md  # 改造方案
    ├── KICKOFF_TASKS.md        # 实施任务
    ├── v2/                     # 技术文档系列
    ├── deprecated/             # 已废弃文档
    └── archived/               # 归档文档
```

## 🎯 设计原则

### 1. 分离关注点

- **CLAUDES.md**: 聚焦"怎么做"（How）
- **CLAUDE.md**: 聚焦"为什么"和"是什么"（Why & What）
- **DOCUMENTATION.md**: 聚焦"在哪找"（Where）

### 2. 角色导向

不同角色有不同的文档入口：
- AI 助手 → CLAUDES.md
- 开发人员 → README.md → CLAUDES.md
- 架构师 → CLAUDE.md
- 新人 → README.md → CLAUDES.md → CLAUDE.md

### 3. 易于维护

- 明确的文档定位，避免内容重复
- 清晰的更新频率说明
- 版本历史记录

### 4. 易于查找

- 文档导航表格
- 按主题索引
- 按角色推荐

## 📈 改进效果

### 对 AI 助手的好处

1. **明确的入口**: CLAUDES.md 作为首选文档
2. **聚焦任务**: 快速找到开发流程和规范
3. **减少干扰**: 不会被详细的业务知识分散注意力
4. **提高效率**: 常用命令和模式一目了然

### 对开发人员的好处

1. **快速上手**: 清晰的文档导航
2. **按需深入**: 可以从简单到详细逐步学习
3. **角色匹配**: 根据角色找到对应文档
4. **便于查找**: 主题索引和角色导向

### 对项目的好处

1. **文档清晰**: 每个文档职责明确
2. **易于维护**: 知道什么时候更新哪个文档
3. **知识沉淀**: 长期和短期知识分离管理
4. **新人友好**: 清晰的学习路径
